## 评审流程（严格按顺序执行）

### 第 1 步：锚定需求文档
阅读 requirements.md，用 1-2 句话总结核心项目和关键模块。确保在评分前清楚 Plan 应该覆盖的范围。

### 第 2 步：逐 Task 依赖链检查
从 T-001 开始逐 Task 验证：每个 Task 声明的依赖是否完整？（引用但未声明的 = 缺失依赖），是否存在循环？

### 第 3 步：逐维度评分
对每个维度执行 3a→3b→3c：
3a. 描述该维度在 Plan 中的表现（1-2 句话）
3b. 与评分表中的 excellent / adequate / poor 校准对比，确定区间
3c. 在区间内赋值。**自检**：描述和分数区间一致吗？不一致 → 回 3a。

### 第 4 步：判定阻塞 vs 建议
阻塞 = 无此修复的 Plan 会导致 AI 工具在某个 Task 遭遇硬阻。不确定时归于建议。
#### 评分一致性校验
如果总分低于 95，回顾 Step 3 中每个扣分维度的评语与分数：
- 评语说"优秀"但给了低分 → 矛盾，回 Step 3 调整
- 评语说"有问题"但给了满分 → 矛盾，回 Step 3 调整
- 每个扣分项的评语必须具体说明扣分原因，不能只写正面评价而给低分
- 不能出现"评分和评语不一致"——这是校验是否通过的关键

### 第 5 步：计算总分
逐维度得分直接加总。passed = (total_score >= 95)。
